Main HTML elements and relations between them

Document type: <!DOCTYPE>

This element represents the document's type, provided so that the computer knows how to interpret** its contents.

Since this is an HTML5 document, the proper way to note it is to write "html" next to the doctype.

Technically, this element is not included in the layer-like structure of the document itself, since it marks its origin. Therefore, it does not have ancestors nor descendants. 

                    
<!DOCTYPE html>

    ...[contents of the document]...

Attributes:

This element does not require any attributes, since their original use is now deprecated [+].

Root element: <html>

This is the top-level element of any HTML document. Therefore, it is referred to as "root element".

It can only contain two elements: a head, <head>, and a body, <body>.

No ancestors are allowed for this element, since it is the main one. 

                    
<!DOCTYPE html>

<html>

    ...[contents of the document]...

</html>

Note: the <html> does not require indentation regarding** the <!DOCTYPE> one.

Attributes:

  • "lang": sets the language of the document.

descendants elements:

  • <head>: contains information about the document (metadata).
  • <body>: defines the contents of the document.

Document's metadata: <head>

This element contains machine-readable information about the document.

Its descendants are elements related to the metadata of the document.

Its only ancestor is the <html> element. 

                    
<!DOCTYPE html>

<html>

    <head>
    
        ...[metadata elements]...
        
    </head>

</html>

Descendant elements:

For structural reasons, this theory block has an independent site, which can be accessed by clicking here.

Sectioning root: <body>

This element contains all the human-readable content of the HTML document (text, images...).

It can contain all the lower-level element-types (according to their corresponding structural rules)

Its only ancestor is the <html> element. 

                    
<!DOCTYPE html>

<html>
    
    <head>
    
        ...[metadata elements]...
    
    </head>
    
    <body>
    
        ...[contents of the document]...
        
    </body>

</html>

Attributes:

  • "id": sets a unique reference for the element along the document.
  • "class": sets a group reference for the elements that contain this attribute under the same name.

Note: [*(some of.../all of....)] these attributes can also be applied to the <body> element.

Content sectioning elements:

Sectioning is mandatory in HTML formatting.

If the content is not properly organized**, the document becomes harder to understand, and might become invalid in terms of normalisation.

Here is a list of the sectioning elements and their functionanity:

<adress>

This element indicates provides entities' contact information.

Example: contact info for a company (URL, e-mail, phone number...).

Allowed content:

All of the Flow elements are allowed, apart from exceptions below:

  • Header (<header>)
  • Titles and title groups (<hgroup>, <h1>, <h2>, <h3>, <h4>, <h5>, <h6>)
  • Self (<adress>)
  • Sectioning content (<section>, <aside>, <nav>)
  • Footer (<footer>)

<article>

This element represents an independent composition in a document. Their elements are interrelated and semantically coherent, therefore able to be distributed independently out of the context of said document.

Examples: daily forecast, comments in a blog...

Allowed content:

All of the Flow elements are allowed.

<aside>

This element represents a portion of a document whose content is indirectly related to the main content.

Example: side notes on newspapers, indirectly related to the article's content, yet not directly.

Allowed content:

All of the Flow elements are allowed.

<footer>

This element contains typical "footer" data of the neares sectioning content or sectioning root element.

Examples: copyright, author credit, company name, links to related pages...

Allowed content:

All of the Flow elements are allowed, apart from exceptions below:

  • Header (<header>)
  • Adress (<adress>)
  • Self (<footer>)

Note: <footer> elements cannot be descendants of <header>, nor <footer> elements and vice versa**, due to obvious reasons.

<header>

This element represents introductory content, typically a group of introductory or navigational aids.

Examples: main logo, author name, search form...

Allowed content:

All of the Flow elements are allowed, apart from exceptions below:

  • Self (<header>)
  • Adress (<adress>)
  • Footer (<footer>)

Note: <header> elements cannot be descendants of <header>, nor <footer> elements and vice versa**, due to obvious reasons.

<hgroup>

This element represents a multi-level heading for a section of a document. It groups a set of section heading elements, explained above.

Examples: complex title-subtitle relations (chapters-subchapters**).

Allowed content:

All of the Flow elements are allowed.

Note: since <hgroup> elements (those that group section heading elements in order to provide a tight title-subtitle relation between them) are now deprecated, they will not be included in this explanation.

<main>

This element represents the dominant content of the <> of a document. It consists of content that is directly related to the central topic of a document, or the central functionality of an application.

Every HTML document itself is an example of the usage of this element. For instance**, the <body> of this webpage contains the explanations about HTML structure.

Allowed content:

All of the Flow elements are allowed.

Note: only one <body> element is allowed per HTML document, for obvious reasons.

<nav>

This element represents a section of a page whose purpose is to provide internal (from the document itself) or external (outer documents) navigation links.

Examples: menus, tables of contents, indexes...

Allowed content:

All of the Flow elements are allowed.

<section>

This element represents a standalone part of the document whose content is interrelated**.

The main content blocks of this webpage are sections, since they can be understood out of context (a title, explanation and properties are given for each element, so that the individual element is comprehensible).

Allowed content:

All of the Flow elements are allowed.

Note: it is recommended to check this brief table of differences between different sectioning elements.

Note: <section> elements cannot be descendants of <adress> ones.
